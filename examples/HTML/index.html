  • Подключение favicon(пример в выше), стилей, скриптов, шрифтов
  • Мета теги
  • Валидация
  • Семантические теги
    • header
    • section
    • footer
    • aside
    • article
    • nav
  • Строчные, блочные теги
  • Теги текста(они семантические если что)
    • h1-h6 => заголовки
    • тег p => параграф
    • address => указание автора статьи(может следовать после тега p)
    • q => цитата
    • blockquote => большая цитата (помещается еще внутрь тега p) атрибут site имеет ссылку на источник, откуда взята эта цитата
    • time => указывает время/дату
  • Типо стилистические теги
    • b - делает текст жирным
    • i - делает текст курсивным
    • tt - делает текст машинным(напечатанный код)
    • s - делает текст зачеркнутым
    • strong- делает текст жирным(выделяем его симантически)
    • em - делает текст курсивным(выделяем его симантически)
    • code - текст выглядит как код
    • pre - предварительное форматирование(например для добавления отступов в коде)
    • small - уменьшает текст(с симантическим смыслом)
    • big - увеличивает текст(с симантическим смыслом)
    • del - зачеркнутый(с симантическим смыслом)
    • ins - подчеркнутый(с симантическим смыслом)
    • sub - нижний индекс(с симантическим смыслом)
    • sup - верхний индекс(с симантическим смыслом)
  • Теги разделители br, hr
  • Списки
    • Маркированный ul(имеет атрибут start, reversed, type)
    • Нумерованный ol(имеет атрибут start, reversed, type)
    • Список определений(обертка - dl, термин - dt, определение - dd)
  • Картинки
    • img, с атрибутом src-путь и alt-альтернативный текст
    • picture включает img и source(внутри атрибут srcset-путь, media-размер)
    • figure(семантических характер) - обертка, внутри img и figcaption-описание для картинки
  • Таблицы
    • table - контейнер для таблицы
    • tr - строка таблицы
    • td - ячейка таблицы
    • caption - заголовок всей таблицы
    • tbody - тело таблицы(строки и ячейки)
    • thead - блок заголовков таблицы
    • tfooter - подвал таблицы(подведение результатов и.т.п)
    • th - ячейка заголовка(вложенность: thead->tr->th)
    • Можно создавать td пустым, это позволит сохранить структуру таблицы(визуально)
    • Атрибут colspan - объединяет ячейки по горизонтали, rowspan - по вертикали
  • Ссылки
    • атрибут download позволяет скачивать файл
    • атрибут _blank откроет ссылку в новой вкладке
    • если в атрибуте href будует tel: - то это позволит сделать вызов номера с ссылки
    • если в атрибуте href будует mailto: - откроет дефолтную прогу на компе для отправки писем
    • если в атрибуте href будует skype: - откроет скайп для звонка
    • якорные ссылки перелистывают страницы к блоку с якорем. У блока должен быть id
  • Кнопки: атрибут type=button => кнопка, submit=>отправить форму reset=>сбросить данные формы; отдельный атрибут autofocus-при загрузке страницы переведет кнопку в фокус, disabled - на кнопку нельзя нажать
  • Полезные теги
    • abbr - аббревиатура, в атрибуте title указывается расшифровка
    • datalist - список вариантов при наборе в текстовом поле. Нужен input с атрибутом list='что-нибудь', а datalist по id свяжется с этим input. Имеет внутри теги option хранящие в атрибуте value варианты выбора
    • meter - вывод значений в каком нибудь известном диапозоне(типо прогресс бар), атрибуты max/min - диапозон, value - само значение, low/high - пределы при которых значение считается низким/высоким и меняется цвет шкалы
    • progress - для отображения прогресса (value - текущее значение; max - максимальное). В отличие от meter не имеет цветового обозначения
    • mark - выделяет цвет в тексте(помечает важную информацию)
    • canvas - для рисования на js
  • Глобальная структура
    • header(может иметь внутри nav) - шапка, заголовок
    • main(может иметь внутри section(может иметь внутри article), header, footer) -основной контент
    • footer(контакты, имя автора, дата, сслыки, копирайт) -подвал
    • aside - неважная информация, реклама
  • Форма - form
      атрибуты:
    • action - файл отвечающий за механизм отправки данных на сервер
    • name - дать имя формы, чтобы отличить от других форм. Используется для отправки данных с конкретной формы
    • autocomplete="on" - заполнить форму уже ранее использованной информацией в этих полях
      еще теги:
    • fieldset - группирует поля формы
    • legend - заголовок в группе fieldset
    • label - устанавливает метку полю атрибут for - для связывания с id input
    • textarea - для ввода большого текста
        атрибуты:
      • rows - количество строк
      • cols - количество столбцов
      • autofocus - устанавливает фокус в данное поле
      • disabled - поле недоступно
      • readonly - доступно только для чтения
      • placeholder - текстовая подсказка
      • name - имя поля
      • maxlength - максимальная длина в символах
      • required - поле обязательно к заполнению
      • select - выбор варианта
      • тег option - опция. Атрибут value идет на сервер при отправке формы
      • тег optgroup - группировка опций
  • input - поле для ввода
      атрибуты:
    • text - поле текста
    • number - только числа min, max, step
    • password - для пароля
    • range - ранжировать что-нибудь(полоса с колесиком)
    • color - выбрать цвет
    • date - задать дату
    • submit - кнопка, если в форме отправит данные на сервер
    • reset - очищает данные, опять же внутри формы
    • select - предлагает из перечня. datalist - можно ввести свое
    • email - для ввода почты
    • url - для ввода url адреса
    • image - кнопка ввиде картинки, отправляет данные
    • date, color, range - очень трудно стилизовать
    • tel - для ввода телефона(на смартфоне откроется клавиатура для ввода номера)
    • search - поле для поиска
    • datetime-local - для установки локального времени
    • month - для выбора месяца
    • time для выбора времени
    • week - для выбора недели
    • hidden - для передачи данных, которые нужно показывать в форме (данные находятся в атрибуте value)
    • file - для выбора файла. атрибут exept указывает какой тип файла загружать
    • radio - выбор одного варианта из нескольких(name нужно указать у всех радио инпутов одинаковый)
    • checkbox - выбор нескольких вариантов из нескольких
  • audio и video
    • тег source атрибут src="файл" type="video/mp4"
    • controls - отобразить панель управления
    • loop - запускает бесконечное воспроизведение видео или аудио
    • autoplay - запуск при загрузке страницы
    • muted - по умолчанию выключить звук
    • poster - указать превью для видео
  • Общие и полезные атрибуты:
    • contenteditable - позволяет редактировать данные тега
    • spellcheck - проверка правописания
    • hidden - скрывает содержимое элемента, но в DOM дереве он существует
    • tabindex - установка порядка фокуса через кнопку tab
    • dir - меняет направление текста внутри тега
    • lang - меняет язык текста внутри тега
